# Design: Structural Breathing / Functional Redundancy (C24)

**Issue**: #177
**Milestone**: M12

## Problem Analysis

当前去 AI 化体系覆盖了词级（黑名单 229 词）、句级（8 种句式模式）、段级（段长 CV、句长 std_dev）的 AI 痕迹检测，但对**章节级结构模板**几乎没有约束。

AI 生成文本的典型结构特征：
- 每个段落精准推进情节，零偏题
- 每句对话直接服务于冲突，无闲聊
- 场景转换高效衔接，无停顿
- 整章"信息效率过高"，读者无消化空间

真人写作的自然特征：
- 偶尔有不服务主线的环境闲描
- 对话中有打趣、抱怨、跑题的"废话"
- 高压场景后有节奏放缓的喘息段
- 部分段落的功能是"让读者消化"而非"推进情节"

## Design Decisions

### D1: 约束定位——ChapterWriter C24

新增 `C24 结构呼吸感`，填补章节级结构盲区：

```
词级: 黑名单 229 词 + replacement_hint
句级: 8 种 AI 句式模式（SP-01 ~ SP-08）
段级: 段长 CV + 句长 std_dev + 标点频率
章级: C24 结构呼吸感（新增）
```

### D2: "功能性停留"定义

功能性停留 = 不直接服务于主线情节推进的叙事片段。类型包括：

| 类型 | 示例 | 功能 |
|------|------|------|
| 环境闲描 | 雨打在铁皮棚上的声音 | 氛围沉浸 |
| 角色闲聊 | "锅里那汤是不是糊了？" | 人物立体感 |
| 感官片段 | 鼻腔里钻进铁锈味 | 物理临场感 |
| 回忆碎片 | 想起小时候被狗追 | 角色深度 |
| 生活细节 | 她哭着还捏着两根葱 | 打破完美感 |

### D3: 密度建议——非硬配额

与技法工具箱（§2.9）设计哲学一致，不设固定配额：
- **建议频率**：每 1000-1500 字至少一处功能性停留
- **非阻断**：QualityJudge 检测到缺失时记为 yellow/suggestion，不自动触发修订
- **上下文敏感**：高潮战斗章可以减少停留，过渡章/日常章可以增加

### D4: 对话冗余度

对话中允许 1-2 句不直接服务冲突推进的"废话"：
- 打趣、抱怨、跑题、自言自语
- 与 C19（对话意图约束）不冲突：废话本身的意图是"敷衍""缓冲""转移"，属于合法意图类型

### D5: QualityJudge 检测——归入 pacing + immersion

不新增独立指标：
- `pacing` 维度：增加"结构呼吸感"评估口径——全章每段都推进情节视为节奏过密
- `immersion` 维度：增加"读者消化空间"评估——无停留段落导致沉浸感下降

### D6: 不在 StyleRefiner 中处理

StyleRefiner 的职责是"不改变语义的表面润色"。功能性停留涉及新增叙事内容，超出 StyleRefiner 的修改权限（Constraint 6 语义不变 + ≤15% 修改量）。只能在 ChapterWriter 生成阶段约束。

## Alternatives Considered

1. **固定配额（每章 ≥N 处停留）**：被否决——与"不固定数量"的设计哲学冲突，且高潮章强制停留会破坏节奏
2. **新增 `structural_redundancy` indicator**：被否决——指标体系已有 13 个，且结构冗余度难以用单一数值量化。归入 pacing/immersion 的定性评估更合理
3. **在 outline 层面约束**：被否决——PlotArchitect 的 outline 粒度是"核心冲突 + 目标"，不适合指定哪里放闲笔。闲笔应由 ChapterWriter 在执行时自然插入
